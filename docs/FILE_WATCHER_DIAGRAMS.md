## Performance Characteristics

### Time Complexity

```
File Change Detection: O(1)
  - Chokidar uses native file system events

Debounce Processing: O(1)
  - Map lookup for existing timer
  - Map insert for new timer

Event Filtering: O(1)
  - Array search in ignore list
  - List size bounded by time window

Redux Update: O(n)
  - n = number of posts
  - Array.find() to locate post
  - Single post update

UI Re-render: O(m)
  - m = number of visible components
  - React's reconciliation
```

### Space Complexity

```
FileWatcherService:
  - ignoredWrites: O(w)
    where w = writes in last 2 seconds
    typically: 1-10 entries

  - debounceTimers: O(f)
    where f = files changed in last 300ms
    typically: 1-5 entries

PostsIpc Cache:
  - O(p) where p = number of posts
  - Bounded by 5-second TTL
  - Invalidated on mutations
```

### Throughput

```
Events per second: ~100-1000
  - Debouncing reduces to: ~3-10 actual updates

File writes per second: ~1-5
  - Batched by middleware debouncing

Memory overhead: ~10-50 KB
  - Ignore list: ~1 KB
  - Timers: ~1 KB
  - Cache: ~10-50 KB (depends on post count)
```
